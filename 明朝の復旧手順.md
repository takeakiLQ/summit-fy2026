### ステップ② GitHub Actions を手動実行 ⭐最重要

ブラウザで開く → **https://github.com/takeakiLQ/summit-fy2026/actions**

1. 左メニューの「**Salesforce → Firestore 自動同期**」をクリック
2. 右上の「**Run workflow**」ボタン → 「Run workflow」（緑）をクリック
3. 約2〜3分待つ
4. 緑チェック✓ が出るまで確認

**ログで以下が見えれば成功:**

```
[3.5/4] 集計サマリーを Firestore に書込（active対象のみで集計）
  active: 4 チーム / 20 メンバー
  集計対象: 579 / 579 件（非アクティブ除外）

  monthly_revenue: 2811 行
  完了: summary/aggregate に書込（財務含む）
```

⚠️ このステップで `summary/aggregate` ドキュメントが Firestore に作られます。これがないとダッシュボードが空表示になります。

## ⚠️ もしステップ②でエラーが出た場合

GitHub Actionsのログを開いて、エラー内容を確認してください。

主なエラーパターン:

| エラー | 対処 |
|---|---|
| `Salesforce 認証失敗` | GitHub Secrets の SF_USERNAME/SF_PASSWORD/SF_SECURITY_TOKEN を確認 |
| `RESOURCE_EXHAUSTED` | まだクォータがリセットされていない。少し待って再実行 |
| `PERMISSION_DENIED` | Firebase の service account 鍵を更新 |
| その他 | エラーメッセージを Claude に共有して相談 |
